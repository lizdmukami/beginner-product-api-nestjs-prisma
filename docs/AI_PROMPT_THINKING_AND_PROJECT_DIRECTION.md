# 🧠 AI Learning Prompt Journal

This journal documents how I used AI to move from uncertainty to clarity while defining and structuring my capstone project. It captures my thought process, doubts, iterations, and final decisions.

---

## 🔹 Prompt 1

**Prompt used:**  
“I have been learning use of ai in programming with moringa school… generate me best performing ideas that will help me get and learn the best and emerge the best student.”

**Intention:**  
To identify a strong project idea that would both help me learn and stand out.

**Feeling at the time:**  
Uncertain. I wanted something impactful but was not sure what direction to take.

**Intended answer:**  
A list of practical, high-performing project ideas.

**AI response (summary):**  
Suggested several project directions including API-based projects, CLI tools, and AI-integrated systems. Emphasized learning and structure over complexity.

**Key takeaway:**  
Focus on projects that demonstrate learning and clarity, not just advanced features.

**Evaluation:**  
Very helpful. It shifted my thinking from “impressive project” to “effective learning project.”

---

## 🔹 Prompt 2

**Prompt used:**  
“I want to do a hardcoded website… full stack ecommerce… API… M-Pesa… minimalistic secure system…”

**Intention:**  
To build a full real-world system combining frontend, backend, and payments.

**Feeling at the time:**  
Ambitious but slightly overwhelmed.

**Intended answer:**  
A structured plan for building a full e-commerce system.

**AI response (summary):**  
Outlined a full stack architecture including Next.js, NestJS, Prisma, PostgreSQL, and M-Pesa. Provided phased development guidance.

**Key takeaway:**  
A full system is possible but must be broken into manageable phases.

**Evaluation:**  
Helpful for vision, but later I realized it was too large for the capstone scope.

---

## 🔹 Prompt 3

**Prompt used:**  
“I want to build the below… specifically want to learn to build an API… kill two birds at the same time.”

**Intention:**  
To refocus the project specifically on API development while still aligning with the e-commerce concept.

**Feeling at the time:**  
More focused but still unsure about scope.

**Intended answer:**  
Validation and refinement of the API-focused approach.

**AI response (summary):**  
Refined the project into a **Product Catalog API**, reducing scope while keeping real-world relevance.

**Key takeaway:**  
Focus on the API layer as the foundation of the system.

**Evaluation:**  
Very helpful. This clarified the direction and made the project realistic.

---

## 🔹 Prompt 4

**Prompt used:**  
“Is what you've given a thing I can build in 7 days considering I only have 2hrs a day?”

**Intention:**  
To assess feasibility within time constraints.

**Feeling at the time:**  
Concerned about overcommitting.

**Intended answer:**  
A realistic breakdown of achievable scope.

**AI response (summary):**  
Confirmed that a reduced API-only version is achievable within 14 hours, while a full system is not.

**Key takeaway:**  
A small, complete system is better than a large unfinished one.

**Evaluation:**  
Very helpful. It helped me avoid scope creep.

To understand what APIs actually do and what output to expect.

**Intention:**  
To elevate the project from a simple submission to a high-quality learning resource.

---

## 🔹 Final Reflection

At the beginning, I was unsure about:
- What project to choose
- How complex it should be
- What the final output would look like

Through AI-assisted prompts, I was able to:
- Narrow down from a full system to a focused API
- Understand how APIs work conceptually
- Align the project with both learning and real-world relevance
- Build confidence in my direction

**Final decision:**  
Build a **Product Catalog API using NestJS, Prisma, and PostgreSQL**, and document the entire process as a beginner-friendly toolkit.

**Confidence level:**  
Clear, focused, and ready to proceed.
